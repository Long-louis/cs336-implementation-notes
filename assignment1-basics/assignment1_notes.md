# Assignment 1 综合笔记

更新时间：2026-04-03

## Encoder 阶段优化（BPE Tokenizer）

1. merge 应用策略优化
原来：在每个 pretoken 上按顺序遍历完整 merges 列表，很多无关规则也会被检查。
现在：初始化时预构建 `merge_ranks`；编码时只检查当前 token 序列真实出现的相邻 pair；每轮选择 rank 最小的可合并 pair 执行合并。
效果：避免大量无效遍历，在大 merges 词表下明显降低计算量。

2. pretoken 缓存优化
原来：同一个 pretoken 每次出现都要重新执行完整 merge。
现在：增加 `_bpe_cache`，以 pretoken bytes 为 key 缓存 token ids；命中缓存直接返回；达到上限后清空防止无界增长。
效果：高频重复片段（常见词、空格前缀词、常见标点）提速明显。

3. 预处理开销优化
原来：编码路径中存在重复构造和重复解析。
现在：GPT-2 pretoken 正则模块级预编译；special token split 正则初始化时编译一次并复用；`bytes_to_id` 初始化一次后编码阶段直接查表。
效果：减少固定开销，提升单次编码效率。

## 编码转换阶段优化（convert_text_to_tokens_npy）

1. 并行任务组织优化
原来：按文件块切分处理，分片管理和中间文件操作成本较高。
现在：使用按行批处理（`batch_lines`）；worker 通过 initializer 初始化一次 tokenizer 并复用；主进程持续投喂批次，worker 返回批次编码后的二进制数据。
效果：降低进程切换与重复初始化开销，多核吞吐更稳定。

2. 写盘路径优化
原来：中间产物和最终写入路径较分散。
现在：先写单个临时 `.tmp.bin`，再通过 memmap 一次性复制到目标 `.npy`；同时清理旧临时产物避免残留干扰。
效果：减少碎片化 I/O 额外成本，输出流程更可控。

3. dtype 默认值优化
原来：默认 `int32`。
现在：默认改为 `uint16`（词表范围满足时使用）。
效果：降低磁盘占用和内存带宽压力，写入与加载更快。
