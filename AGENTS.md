# CS336 课程仓库代理工作指南（根目录共享版）

## 1) 适用范围
- 本文件适用于整个 `cs336/` 仓库（含 `assignment1-basics/` 与后续 `assignment*` 目录）。
- 若某个 assignment 内存在更细粒度约束（如本地 `AGENTS.md`、handout、README），以该 assignment 目录内规则为准。

## 2) 总体原则
- 先对齐当前作业 handout 的 Problem 要求、评分点和约束，再做实现。
- 优先复用项目已有模块与脚本，避免平行重复实现。
- 变更保持最小化，尽量只改与当前任务直接相关的文件。
- 先实现再验证：每次改动后运行对应最小测试，再按需扩大验证范围。

## 3) 仓库组织约定
- 根目录用于课程级资源：跨作业脚本、同步工具、全局配置。
- 各 `assignment*` 目录用于作业级实现：代码、测试、数据说明、作业文档。
- 不把临时产物提交到 Git：训练日志、checkpoint、缓存、临时数据文件。

## 4) 代码与测试工作流
- 实现入口优先从 assignment 内 `tests/adapters.py` 或等价适配层反推。
- 对模型/数值相关实现，优先保证函数签名与张量形状约束正确。
- 测试顺序建议：
  1) 与改动最相关的单测
  2) 同模块测试
  3) assignment 全量测试（必要时）

## 5) 环境与依赖
- Python 项目优先使用 `uv` 管理环境与依赖。

## 6) 远程训练与同步
- 代码同步优先 Git（可追踪、可回滚）。
- 数据/checkpoint/日志优先 `rsync` 增量同步。
- 建议使用根目录 `scripts/` 中的同步脚本进行上行代码和下行产物管理。
